# Groq как Vision-бэкенд для расп��знавания чеков

## Проблема

Mistral Small 3.1 24B (через Polza или self-hosted) плохо распознаёт чеки — мелкий текст, перекосы, плохой контраст. Polza не имеет качественных Vision-моделей для OCR чеков.

## Поиск решения (2 июня 2026)

Были рассмотрены варианты:

### Self-hosted (на поезд, CPU) — ❌ не взлетает
- Qwen2.5-VL 7B — минимум 7B параметров, на CPU чек за минуты
- Pixtral 12B — ещё тяжелее
- Gemma 3 4B — терпимо, но качество под вопросом
- PaddleOCR-VL (0.9B) — слабая для чеков
- **Вывод:** на CPU качественный Vision не поднять

### Gemini 2.5 Flash — ❌ заблокирован в РФ
- Бесплатно 1500 запросов/день, лучшее OCR-качество
- Не работает из России

### ✅ Groq + Llama 4 Scout — РАБОТАЕТ
- **Бесплатно:** 30 RPM, 14 400 RPD
- **Скорость:** молниеносно (Groq на собственном железе)
- **OCR:** среднее (читает магазин/сумму/дату хорошо, товары может галлюцинировать)
- **Работает из РФ:** да
- **Кредитка:** не нужна

### OpenRouter + Qwen2.5-VL-7B — ❌ не работает
- OpenRouter в РФ заблокирован
- Polza (наш провайдер) как прокси не имеет Vision-моделей для чеков

## Итоговое решение

1. **Groq** подключён как Vision-бэкенд для распознавания чеков
2. Модель: `meta-llama/llama-4-scout-17b-16e-instruct` через API api.groq.com
3. Ключ Groq хранится в `/root/.hermes/.env`
4. **Ограничение:** товары на чеке могут галлюцинироваться — нужно перепроверять (правило: «не додумывать, если не уверена — спросить»)

## Уроки

1. Polza не подходит для Vision-задач — только текстовые модели
2. Groq — бесплатный и быстрый, но OCR качества уступает Gemini
3. Для надёжности можно комбинировать: Groq (основной) + OCR.space (резерв, 25 000 запросов/мес бесплатно)
