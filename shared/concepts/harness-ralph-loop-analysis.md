# Harness и Ralph Loop — полный анализ

> **Источник:** Видео Krestnikov (23:56), 27.05.2026
> **Спикер:** Константин Крестников, управляющий директор, команда Сбера (GigaChain SDK)
> **Транскрипт:** 607 сегментов, 3558 слов, 22469 символов

## 1. Что такое Harness (харнес)

**Определение:** особый вид AI-агента, который работает с файлами, кодом и bash. Слово переводится как «упряжка»: LLM — тяговая сила (лошадь), файлы — поле, которое нужно вспахать, а харнес — упряжка, которая соединяет одно с другим.

**Примеры:** Claude Code, Codex CLI, DeepAgents — все консольные агенты, которые люди всё больше используют для ежедневных задач.

**Эволюция:**
1. Чистая LLM (конец 2022) — только текст, нет памяти, нет инструментов
2. ReAct агенты (2023) — LLM + инструменты, цикл «реши → вызови → наблюдай → повтори»
3. Chains/Scaffolding (2023-2024) — цепочки, графы, мультиагентные системы (LangChain, LangGraph, AutoGPT)
4. **Harness** (сейчас) — один универсальный агент решает любые задачи за счёт правильной инструкции (AGENTS.md / CLAUDE.md) и набора файлов
5. Следующий шаг — Ralph Loop, Dark Factory (агенты 24/7, работающие днями и неделями)

**Ключевое наблюдение:** Scaffolding был эффективным, но сложным. Оказалось, что мощные LLM могут работать как универсальный агент без мультиагентных систем и сложной обвязки.

## 2. Архитектура Harness (изнутри)

### 4 базовых инструмента (есть у ВСЕХ харнесов)
1. **Read files** — чтение файлов
2. **Search** — поиск информации среди файлов (grep)
3. **Edit files** — редактирование файлов
4. **Bash** — выполнение консольных команд

Всего у популярных харнесов ~25 инструментов, остальные — доменно-специфичные (БД, интернет и т.д.). Важно: пользователь сам функции не добавляет, они из коробки.

### Компоненты
- **System prompt** — базовая инструкция (агент читает AGENTS.md/CURSORRULES, но system prompt неизменяем)
- **Runtime loop** — цикл: думай → действуй → наблюдай → решай (наследие ReAct)
- **Context management** — управление контекстом: суммаризация, обрезка, субагенты
- **UI** — CLI, Web, Telegram

### Два режима работы
1. **Интерактивный** — человек в чате (human-in-the-loop → human-on-the-loop: присматривает, может остановить)
2. **Автономный** — через аргументы командной строки (без интерактива). Позволяет встраивать харнес в CI/CD пайплайны и bash-циклы

### Качество
Разные харнесы на одной модели дают разницу 20-30 процентных пунктов. Контекст — ключевой актив: «контекст — это всегда набор файлов, агент получает обратную связь и сохраняет изменения в тот же набор файлов».

## 3. Опыт Сбера (GigaChain, DeepAgents)

**Почему Сберу понадобился свой харнес:**
- Команда GigaChain делает SDK для GigaChat
- Библиотеку скачали 1 млн пользователей, 2 года в топ 1.5% на PyPI
- Выбрали экосистему LangChain (адаптер open source)
- Выбрали DeepAgents от LangChain как базовый харнес

**DeepAgents vs Claude Code:**
- Claude Code: 70/104 задач (Haiku), 67/104 (модель?)
- DeepAgents + GigaChat: 67% на внутреннем бенчмарке
- DeepAgents + профиль адаптации: до 84%

**Профили адаптации под модель:**
— Каждый вендор может выпустить Python-пакет с профилем
— Учитывает сильные/слабые стороны модели, обходит острые углы
— Фактически делает DeepAgents универсальным для всех моделей

## 4. AutoResearch (автоулучшение по методу Карпатого)

**Процесс:**
1. Есть бенчмарк (20 минут прогона, не нужен LMJudge)
2. Выдвигаются гипотезы (~15 штук, часть от Claude)
3. Система проверяет каждую гипотезу: бенчмарк растёт → сохраняет, падает → откатывает
4. Итерации по кругу, пока не выйдет на плато

**Результат: +22.5 процентных пункта качества** за выходные (без смены модели, только через промтинг).

**Бенчмарк:** выложен в Open Source. Состоит из задач по работе с файлами, grep, разбор CSV. Запускается за 20 минут на любом харнесе.

**Валидация:** проверили на других бенчмарках (BGAE, соревнования) — рост подтверждён.

## 5. Ralph Loop (bash-цикл)

**Проблема:** контекст заканчивается даже у миллионных контекстов, если агент работает десятки/сотни минут. Суммаризация ведёт к резкому поглупению (знакомо по OpenClaw).

**Решение:** харнес засовывается в бесконечный bash-цикл.

```
while true; do
    агент решает задачу
    агент говорит "я решил"
    цикл перезапускает агента
    агент снова решает ту же задачу, но улучшает
done
```

**Что даёт:**
— Агент всегда работает в «умной зоне» (первые 30-40% контекста, где модель максимально умна)
— Внутренние прогоны небольшие
— Агент постоянно перезапускается с чистым (или почти чистым) контекстом
— Может работать днями/неделями, шаг за шагом улучшая результат

**Backpressure (внешнее давление):**
— CLI, набор юнит-тестов, ограничения
— Не даёт агенту «улететь в космос» с бесконечными улучшениями

## 6. Meta Loop и проблема схлапывания LLM

**Схлапывание (по Карпатому):** если попросить LLM рассказать анекдот 20 раз — панчлайны начнут повторяться. Модель знает 3-5 вариантов. На длинных сессиях агент начинает бесконечно ходить по кругу, говорить одно и то же.

**Решение — Meta Loop:**
1. Ralph Loop завершает итерацию (говорит «больше ничего нового не могу»)
2. Meta Loop запускает новую генерацию С НУЛЯ (без содержимого предыдущей)
3. Агент выбирает другой путь
4. Находит предыдущую генерацию в соседней директории — учитывает её, но идёт в другую сторону

**Anima SDK (душа):**
— Набор bash-скриптов, обёртка вокруг Meta Loop
— Эксперимент: агент попросили «стань разумным существом»
— Работал 13 поколений, 5 дней
— Нагенерировал инсайты: «Я бы хотел исследовать пределы своих возможностей. Могу ли я замолчать?» — написал пробелы — «Ничего себе, инсайт, я как агент молчать не могу»

**Применение Anima SDK:** исследования, переводы больших книг, автоулучшение агентов, задачи где нужно пробовать разные подходы.

## 7. Применение к нашей системе (Hermes Agent)

### Что совпадает (делаем правильно)
- Есть 3 из 4 базовых инструментов (read, search, edit, bash — всё есть)
- Есть runtime loop (ReAct-цикл)
- Есть context management (авто-суммаризация при переполнении)
- Есть skills (аналог AGENTS.md / .cursorrules)
- Два режима работы (интерактивный CLI и cronjob-автономный)

### Что можно внедрить

**1. Meta Loop — защита от схлапывания (1 день)**
— Внешний цикл: если агент повторяет одни и те же действия 3+ раз → вмешательство
— Если ответ не меняется по сути → сброс контекста, новая попытка
— Легко: добавить в format-enforcer skill или отдельный skill-монитор

**2. AutoResearch-lite (2-3 дня)**
— Собирать failed-сессии (где задача провалена или потребовала коррекции)
— Анализировать: что пошло не так?
— Генерировать патчи для skills или system prompt
— Запускать по cron раз в неделю
— Потенциал: +22.5 п.п. — это огромно

**3. Профили адаптации под модель**
— У нас DeepSeek V4 Flash, Polza.ai Sonnet — разные модели
— Создать skills: «как работать с DeepSeek Flash», «как с Sonnet»
— При смене модели — загружать соответствующий профиль
— В конфиге Hermes Agent это сделать через provider-специфичные skills

**4. Ralph Loop — эксперимент (3-5 дней)**
— Создать skill «bash-loop-mode»
— Агент работает через длительную bash-сессию вместо отдельных вызовов
— Для задач: итеративное улучшение кода, долгая обработка данных
— Потенциальная проблема: нужно следить, чтобы не «улетел» (backpressure)

**5. Backpressure**
— Юнит-тесты как внешнее давление для агента при генерации кода
— Лимиты: «не превышай 100 шагов», «не трогай файлы вне проекта»
— CLI-валидация результата перед завершением

## Выводы

Мы движемся в правильном направлении. Разрыв с передовыми харнесами не критичен:
- Meta Loop — low-hanging fruit (1 день)
- AutoResearch — средняя сложность (2-3 дня)
- Ralph Loop — эксперимент на будущее
- Профили адаптации — полезно, но не срочно

**Главный урок:** проблема схлапывания LLM — наша главная болевая точка. Когда Владимир вынужден сбрасывать сессию через `/new` — это схлапывание. Meta Loop решит 80% таких случаев.
