# Slash-команды Hermes — платформенные различия

## Определение

Slash-команды (слэш-команды) Hermes — унифицированный набор текстовых команд, начинающихся с `/`, которые работают через gateway на всех платформах: Telegram, Max Messenger, Discord, CLI. Команды обрабатываются централизованно, независимо от того, с какой платформы они отправлены.

## Стандартный набор команд

| Команда | Назначение |
|---------|-----------|
| `/help` | Справка по командам |
| `/restart` | Перезапуск сессии |
| `/approve` | Подтверждение действия |
| `/deny` | Отклонение действия |
| `/update` | Обновление агента |
| `/sethome` | Установка домашней директории |
| `/topic` | Смена темы сессии |
| `/platforms` | Список платформ |

## Платформенные различия

### Telegram (полная поддержка)
- **Автодополнение:** Да — боты регистрируют команды через Bot API, пользователь видит подсказки при вводе `/`.
- **Inline-кнопки:** Есть — для `clarify` (уточнение) отображаются кнопки под сообщением.
- **Режим работы:** Webhook (рекомендуемый).

### Max Messenger (ограниченная)
- **Автодополнение:** НЕТ — команды нужно вводить вручную целиком.
- **Inline-кнопки:** НЕТ — Max не поддерживает inline-интерфейс для ботов.
- **Режим работы:** Long-polling (текущий). Webhook рекомендуется, но не реализован.
- **Медиа:** Поддерживает фото, аудио, видео, файлы, стикеры через `MessageType`. Голосовые через long-poll не доставляются (ограничение API).
- **Слэш-команды не предназначены для отправки медиа** — только текст.

### Discord
- Полная поддержка с регистрацией команд через Discord API.
- Адаптер имеет собственную логику регистрации/перезаписи команд.

## Архитектура обработки

- `gateway/slash_access.py` — система контроля доступа: разрешение/запрет команд по платформам и пользователям (allow_admin_from, user_allowed_commands).
- Каждая платформа (Telegram, Max, Discord, Slack) имеет свой адаптер, но слэш-команды обрабатываются единым gateway.
- Адаптер Max не имеет собственной реализации слэш-команд — он наследует общую gateway-обработку.

## Как добавить автодополнение на Max (гипотетически)

1. **На стороне API Max:** требуется специальный метод для регистрации команд (если его нет — невозможно).
2. **На стороне клиента:** реализовать автодополнение в web-чате дашборда (chat.penoizol-chernozem.ru).
3. **Ограничение:** задача не решена, автодополнения на Max нет (май 2026).
