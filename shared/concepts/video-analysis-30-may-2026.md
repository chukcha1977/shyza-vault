**Канал:** David Ondrej | **Длит.:** 32:56 | **Язык:** EN

## Видео 6: Pangolin — зачем он нужен твоему homelab
**Ссылка:** https://youtube.com/watch?v=EfiLQ6cWDMY
**Канал:** Русский (Stealtho-style) | **Длит.:** 28:35 | **Язык:** RU

**Резюме:** Полный гайд по Pangolin — open-source приложению для self-hosted доступа к домашним сервисам извне без открытия портов. Комбинирует обратный прокси (Traefik), WireGuard VPN туннели и Zero Trust Network (аналог Cloudflare Tunnels).

**Архитектура:**
- **Pangolin** (мозг) — управляет всеми компонентами
- **Traefik** (маршрутизатор) — обратный прокси, SSL/TLS сертификаты (Let's Encrypt через HTTP challenge)
- **ForwardAuth** (middleware) — авторизация/аутентификация пользователей перед доступом
- **Jwthelper** (JWT helper) — управление WireGuard туннелями
- **Edge** (клиент) — легковесный WireGuard туннель на домашней машине, инициализирует соединение наружу

**Ключевые идеи:**
1. **Zero портов на роутере** — не нужно открывать порты, пробрасывать, бороться с двойным NAT
2. Безопасность: даже если VPS скомпрометирован — данные в homelab не тронуты
3. Установка одной командой через официальный скрипт (Docker Compose)
4. Бесплатно для физиков и бизнеса с доходом < $100K/год (Enterprise Edition)
5. WireGuard работает в РФ (пока не забанен)
6. Геоблок по MaxMind GeoIP2 — можно ограничить по странам
7. Поддержка внешних IDP (Authelia, Authentik) через Enterprise лицензию
8. Pincode-защита для быстрых инвайтов
9. Русский перевод машинный — лучше использовать английский интерфейс

**Требования:** VPS (рекомендовано 2 CPU/4GB RAM/20GB SSD), доменное имя, открытые порты 80/443 на VPS
**Пример:** Развёрнут Docmost (замена Portainer) через Pangolin-туннель с SSL от Let's Encrypt

**Применимость:** Альтернатива Cloudflare Tunnels для РФ. Если понадобится безопасный доступ к нашим сервисам извне — Pangolin решает без Cloudflare.

## Видео 7: Hermes Agent Update v0.15 "Velocity"
**Ссылка:** https://youtube.com/watch?v=GL2FhteoPBA
**Канал:** Ron (AI content) | **Длит.:** 13:18 | **Язык:** EN

**Резюме:** Обзор крупного обновления Hermes Agent до v0.15 Velocity. Полный рефакторинг ядра, ускорение запуска, Kanban Swarm оркестрация, безопасность, Bitwarden.

**Ключевые изменения:**
1. **Рефакторинг ядра:** main loop с 16,000 → 3,800 строк, разбит на 14 модулей. Совместимость с предыдущими конфигами сохраняется
2. **Скорость:** `hermes --version` с 700ms → 258ms. Затраты на function call снижены с 399K → 213K вызовов
3. **Session Search:** с 90 секунд → 20 миллисекунд (4500× быстрее). Больше не требует LLM. Можно возобновлять сессии и видеть полный текст
4. **Kanban Swarm:** новая команда `hermes kanban swarm`. Граф с root-задачей, параллельные workers, verifier, synthesizer. Work tree per task — каждая задача в своей ветке/директории. TTL (таймаут зависших задач), retry fingerprinting (защита от повторения одних ошибок), stale task detection с respawn guards
5. **Bitwarden интеграция:** один bootstrap token, чистая ротация секретов
6. **Безопасность:** блокировка prompt injection на 3 choke points (tool output, recalled memory, stored skills) + 15 новых threat patterns + security guidance plugin
7. **MCP Catalog**, Notify (23-я платформа), Koren для генерации изображений
8. **Hotfix v0.15.1:** исправлен infinite reload dashboard в Docker, Kanban worker, MCP команды
9. **Skill bundles** и **TUI session orchestrator** (переключение между живыми сессиями)

**Что это значит для нас:**
- Наш Hermes нужно обновить до v0.15 — радикальное ускорение session search (я им пользуюсь) и Kanban Swarm (можно для параллельных задач)
- Session search теперь 20ms вместо 90s — отличная новость, я смогу быстрее находить прошлые контексты
- Security update критичен — мы храним API-ключи в конфигах
- Kanban Swarm потенциально решает проблему параллельных задач (наши 3 агента?)
