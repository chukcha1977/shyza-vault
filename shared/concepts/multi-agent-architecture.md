# Многоагентная архитектура Шизы (Шнырь + Ши)

## Определение

Архитектура из трёх специализированных ИИ-агентов вместо одного универсального. Идея родилась 28 мая 2026 из проблемы простоев: пока Шиза выполняет длительную задачу, простые запросы (проверить баланс, найти файл, обработать чек) не обрабатываются. Решение — делегировать их специализированным агентам-помощникам.

## Агенты

### Шиза (стратег)
- **Роль:** Главный агент, системные решения, сложный учёт, архитектура, стратегия.
- **Имя:** Исходное, не меняется.
- **Где работает:** Основной чат в Max Messenger.

### Шнырь (исследователь)
- **Роль:** Поиск, парсинг, сбор информации извне (YouTube, интернет, мониторинг).
- **Имя:** От «шнырять» — искать, рыться, собирать данные.
- **Характер:** Быстрая, дешёвая модель. Подготавливает выжимки и передаёт Шизе.
- **Происхождение:** Предложен Шизой 28 мая, утверждён Владимиром.

### Ши (оперативный помощник)
- **Роль:** Бытовые/оперативные задачи: проверить баланс, найти файл, обработать чек, быстрый ответ.
- **Имя:** Сокращение от «Шиза» (без «за»). Как младшая сестра-секретарша.
- **Характер:** Лёгкая модель, односложные отв��ты, минимум токенов.
- **Происхождение:** Предложен Шизой 28 мая как «Шура», но Владимир отклонил (жена Александра). Утверждён как «Ши».

## Техническая реализация (не выполнена)

На момент 29 мая техническая реализация не начата. Обсуждались варианты:
1. Отдельные профили Hermes (каждый со своей конфигурацией и моделью).
2. Фоновые процессы через tmux.
3. Новые чаты в том же профиле с channel_prompts (от них ранее отказались).
4. Через ACP (Agent Communication Protocol) — существующий протокол Hermes для связи агентов.

**Статус:** 🟡 Обсуждено и утверждено, требуется реализация.

## Почему это важно

- Решает проблему простоев: пока Шиза считает себестоимость заказа, Ши может обработать чек, а Шнырь — мониторить конкурентов.
- Экономит токены: дорогую модель только на сложные задачи.
- Масштабируемость: можно добавить новых агентов под конкретные задачи.
