## Ключевые идеи

1. **AI OS > модель.** Claude/GPT/Grok — сменные ядра. Операционная система на markdown (агенты, навыки, воркфлоу, правила, знания) — вечна. Меняешь модель — OS остаётся.

2. **Два типа данных для LLM:**
   - Количественные: SQL, BI, цифры
   - Качественные: текст, стратегии, гайдлайны, брендбуки, визуалы
   - Качественные — важнее, но их сложнее хранить для AI

3. **Граф знаний vs иерархия папок:**
   - Файловая система: one-to-many, AI тормозит при поиске
   - Граф: one-to-many-to-many, AI «летает» и находит связи, невидимые человеку
   - Пример: Klaviyo draft → проект Memorial Day → инструмент email → отдел маркетинга → сезонная акция → voice компании — 6 связей вместо одной папки

4. **Индексация с резюме:**
   - AI может прочитать ~500 заметок до ответа
   - У каждой заметки — краткое резюме в индексе
   - AI читает индексы → находит нужное → углубляется → ответ за секунды

5. **Infinite Brain (опенсорс с 10 июня 2026):**
   - Всё в markdown
   - Сущности: агенты (persona + capability), навыки, воркфлоу, правила, инструменты, знания, проекты, outputs
   - Три бесплатных инструмента чтения: Obsidian (граф), n8n (workflow), Paperclip (агенты)
   - Claude Code / Codex — инструменты для сборки сверху

6. **Масштабирование снизу вверх:**
   - Каждый сотрудник → свой репозиторий
   - Лучшие артефакты → отдельский репозиторий
   - Отделы → компания
   - AI Architect сервис ревьюит и апгрейдит
