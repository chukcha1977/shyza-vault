# SCHEMA — ДНК системы знаний

|> AGENTS.md читать ПЕРВЫМ при старте. Этот файл — ДНК системы, заглядывать при непонимании архитектуры.

## Структура папок

```
brain/           — стратегический контекст (about, values, projects). Читать при старте.
raw/             — сырые данные. Immutable. Только добавление, не редактировать.
sources/         — внешние источники (статьи, видео, книги с выводами и применимостью).
concepts/        — обработанные знания (domains: penoizol, dashboard, website, hermes, business).
entities/        — сущности: люди, компании, организации.
_templates/      — шаблоны для новых записей (concept, entity, source, project).
index.md         — карта системы. Читать второй при старте.
log.md           — хронология операций.
hot.md           — горячий кэш между сессиями. Автообновление при завершении сессии.
SCHEMA.md        — этот файл. ДНК системы.
```

## Workflow

### SessionStart (каждый запуск)
1. Прочитать AGENTS.md — правила навигации (дешевле SCHEMA.md)
2. Прочитать hot.md — что было в прошлый раз
3. Прочитать index.md — карта знаний
4. Только если нужно углубиться — читать конкретные статьи из index
5. brain/about.md и brain/values.md — только если не знаешь, кто Владимир

### Ingest (обработка нового сырья)
1. Найти новые файлы в raw/
2. Прочитать содержимое
3. Создать заметки в concepts/, sources/ или entities/
4. Обновить index.md
5. Записать в log.md

### Lint (еженедельная проверка)
1. Все ли ссылки в index.md ведут на существующие файлы
2. Есть ли сироты (файлы без обратной ссылки)
3. Актуален ли hot.md

### SessionEnd (завершение сессии)
1. Сохранить ключевые выводы в hot.md
2. Если много нового — записать в log.md

### Правила записей

- Каждая заметка: YAML frontmatter (type, title, created, updated, tags, related, sources)
- Ссылаться на связанные заметки викиссылками
- Новые статьи — в index.md под правильный раздел
- Каждое действие — запись в log.md

## Типы заметок

| type | Папка | Шаблон |
|------|-------|--------|

| entity | entities/ | _templates/entity.md |
| source | sources/ | _templates/source.md |

## Питфоллы

- Не модифицировать raw/ файлы. Они immutable.
- Не дублировать данные из MariaDB в vault. В vault — анализ, в БД — цифры.
- Перед записью любого файла — показать Владимиру и ждать «да».
